    Detailed Description

    Extends QWebMethod with some generic constructors and synchronous message sending.

    Subclass of QWebMethod, contains some generic methods for sending messages and various additional convenience constructors. Can be used both synchronously (through static sendMessage() method), or asynchronously (indicates, when reply is ready by emitting a replyReady() signal).

    Member Function Documentation

    QWebServiceMethod::QWebServiceMethod ( QObject * parent = 0 )

    Constructs web method object with parent. Requires specifying other parameters later. If you use that constructor, you can probably go on and use the base QWebMethod class.

    QWebServiceMethod::QWebServiceMethod ( Protocol protocol = Soap12, HttpMethod httpMethod = Post, QObject * parent = 0 )

    A constructor that takes in protocol information, httpMethod to use, and parent to satisfy QObject requirements. All other data has to be set using setter methods.

    QWebServiceMethod::QWebServiceMethod ( const QUrl & hostUrl, const QString & methodName, Protocol protocol = Soap12, HttpMethod method = Post, QObject * parent = 0 )

    Constructs the message using hostUrl, methodName, parent, protocol (which defaults to soap12), and method (which defaults to POST). Requires params to be specified later.

    See also setParameters(), setProtocol(), and sendMessage().

    Constructs the message using host, methodName, parent, protocol (which defaults to soap12), and httpMethod (which defaults to POST). Requires params to be specified later.

    See also setParameters(), setProtocol(), and sendMessage().

    QWebServiceMethod::QWebServiceMethod ( const QString & host, const QString & methodName, const QString & targetNamespace, const QMap<QString, QVariant> & params, Protocol protocol = Soap12, HttpMethod httpMethod = Post, QObject * parent = 0 )

    Constructs the message using host, methodName, targetNamespace, parent, protocol (which defaults to soap12), and httpMethod (which defaults to POST). This constructor also takes message parameters (params). Does not require specifying any more information, but you still need to manually send the message using sendMessage() (without any arguments, or else - if you want to change ones specified here).

    See also sendMessage() and setProtocol().

    bool QWebServiceMethod::sendMessage ( const QMap<QString, QVariant> & params )

    This function overloads sendMessage().

    Sends the message asynchronously using parameters specified in params.
