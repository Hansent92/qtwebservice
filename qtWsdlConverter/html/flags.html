    Detailed Description

    Holds all important flags used by converter.

    Currently, every setting is held in a different enum, and all variables are public. In the future, flags will probably be unified into one/ two enums, thus enabling easy OR switching.

    Member Type Documentation

    enum Flags::BuildSystem

    This enum type specifies the build system to use:

    ConstantValueDescription
    Flags::qmake0qmake will be used.
    Flags::cmake1cmake will be used.
    Flags::scons2scons will be used.
    Flags::noBuildSystem3No build system files will be created.

    enum Flags::Mode

    This enum type specifies code creation mode:

    ConstantValueDescription
    Flags::fullMode0All enums and variables are copied, enabling some flexibility in use.
    Flags::debugMode1fullMode + debug messages.
    Flags::compactMode2Only most needed methods, variables and enums are preserved. Code is as small, as possible, at the expense of loosing flexibility.

    enum Flags::Structure

    This enum type specifies the filesystem structure (the way files will be written on disk):

    ConstantValueDescription
    Flags::standardStructure0Headers will be placed in <dir>/headers, sources in <dir>/sources, build system and .pro files in <dir>/.
    Flags::noMessagesStructure1Converter will not create messages as separate classes, but use QSoapMessage.
    Flags::allInOneDirStructure2All generated files will be stored in one folder.

    enum Flags::Synchronousness

    This enum type specifies the way messages will be sent and received:

    ConstantValueDescription
    Flags::synchronous0Web methods will wait for reply and return with data.
    Flags::asynchronous1Web methods will return instantly. Data availability will be announced by emitting a signal.
