    Detailed Description

    Holds all important flags used by converter.

    All application's switches are unified thus enabling easy OR switching. The only exception is the --force switch, which is held as a separate bool variable.

    All entries in single sections (with an exception of build system related switches) are exclusive - you can use only one at a time (the lates one specified is used). For example, if you use --http and --json, only JSON flag will be set. Sections are specified in enum documentation (the table below).

    Member Type Documentation

    enum Flags::Option
    flags Flags::Options

    This enum type specifies code creation mode. Sections are indicated in brackets:

    Mode:

    ConstantValueDescription
    Flags::fullMode0x00001(mode) All enums and variables are copied, enabling some flexibility in use.
    Flags::debugMode0x00002(mode) fullMode + debug messages.
    Flags::compactMode0x00004(mode) Only most needed methods, variables and enums are preserved. Code is as small, as possible, at the expense of loosing flexibility.
    Flags::synchronous0x00010(synchronousness) Web methods will wait for reply and return with data.
    Flags::asynchronous0x00020(synchronousness) Web methods will return instantly. Data availability will be announced by emitting a signal.
    Flags::standardStructure0x00100(structure) Headers will be placed in <dir>/headers, sources in <dir>/sources, build system and .pro files in <dir>/.
    Flags::noMessagesStructure0x00200(structure) Converter will not create messages as separate classes, but use QWebMethod.
    Flags::allInOneDirStructure0x00400(structure) All generated files will be stored in one folder.
    Flags::qmake0x01000(build system) qmake will be used.
    Flags::cmake0x02000(build system) cmake will be used.
    Flags::scons0x04000(build system) scons will be used.
    Flags::noBuildSystem0x08000(build system) No build system files will be created.
    Flags::http0x10000(protocol) http protocol will be used.
    Flags::soap100x20000(protocol) SOAP 1.0 protocol will be used.
    Flags::soap120x40000(protocol) SOAP 1.2 protocol will be used.
    Flags::soap0x60000(protocol) Internall wrapper for all SOAP protocols. When used in app, will default to SOAP 1.2
    Flags::json0x80000(protocol) JSON protocol will be used.

    The Options type is a typedef for QFlags<Option>. It stores an OR combination of Option values.

    Member Function Documentation

    Flags::Flags ( Options options = Options( fullMode | synchronous | standardStructure | qmake | soap12 ), bool forced = false )

    Constructs the Flags object. All params are optional. If you want to set non-default suffixes, use setObjSuffix() and setMsgSuffix().

    Default values:

    • options :
      • fullMode
      • synchronous
      • standardStructure
      • soap12
      • qmake
    • forced = false

    See also setFlags().

    void Flags::resetFlags ()

    Resets all flags to default values. This does not apply to '--forced'.

    void Flags::resetFlags ( Options whatToReset )

    Resets (zeroes) flags set to 1 in whatToReset. This does not apply to '--forced'.

    void Flags::setFlags ( Options options )

    Does OR combination of options with those already set.

    See also flags().
