Detailed Description

Holds all important flags used by converter.

All application's switches are unified thus enabling easy OR switching. The only exception is the –force switch, which is held as a separate bool variable.

All entries in single sections (with an exception of build system related switches) are exclusive - you can use only one at a time (the lates one specified is used). For example, if you use –http and –json, only JSON flag will be set. Sections are specified in enum documentation (the table below).

Member Enumeration Documentation

enum Flags::HttpMethod

This enum stores HTTP method chosen by user. Those options cannot be combined!

Post HTTP POST will be used. Get HTTP GET will be used. Put HTTP PUT will be used. Delete HTTP DELETE will be used.

enum Flags::Option

This enum type specifies code creation mode. Sections are indicated in brackets:

FullMode (mode) All enums and variables are copied, enabling some flexibility in use. DebugMode (mode) fullMode + debug messages. CompactMode (mode) Only most needed methods, variables and enums are preserved. Code is as small, as possible, at the expense of loosing flexibility. Subclass converter creates web methods by subclassing QWebMethod (this option requires QWebService library to be present). Synchronous (synchronousness) Web methods will wait for reply and return with data. Asynchronous (synchronousness) Web methods will return instantly. Data availability will be announced by emitting a signal. StandardStructure (structure) Headers will be placed in <dir>/headers, sources in <dir>/sources, build system and .pro files in <dir>/. NoMessagesStructure (structure) Converter will not create web methods as separate classes, but use QWebMethod (this option requires QWebService library to be present). AllInOneDirStructure (structure) All generated files will be stored in one folder. Qmake (build system) qmake will be used. Cmake (build system) cmake will be used. Scons (build system) scons will be used. NoBuildSystem (build system) No build system files will be created. Http (protocol) http protocol will be used. Soap10 (protocol) SOAP 1.0 protocol will be used. Soap12 (protocol) SOAP 1.2 protocol will be used. Soap (protocol) Internall wrapper for all SOAP protocols. When used in app, will default to SOAP 1.2 Json (protocol) JSON protocol will be used. Xml (protocol) XML protocol (non-SOAP) will be used. Rest (non-exclusive, protocol) Code for communicating with RESTful WSes will be generted. This can be combined with any other protocol flag.
