    Detailed Description

    Holds all important flags used by converter.

    All application's switches are unified thus enabling easy OR switching. The only exception is the --force switch, which is held as a separate bool variable.

    All entries in single sections (with an exception of build system related switches) are exclusive - you can use only one at a time (the lates one specified is used). For example, if you use --http and --json, only JSON flag will be set. Sections are specified in enum documentation (the table below).

    Member Type Documentation

    enum Flags::HttpMethod

    This enum stores HTTP method chosen by user. Those options cannot be combined!

    ConstantValueDescription
    Flags::Post0x1HTTP POST will be used.
    Flags::Get0x2HTTP GET will be used.
    Flags::Put0x4HTTP PUT will be used.
    Flags::Delete0x8HTTP DELETE will be used.

    enum Flags::Option
    flags Flags::Options

    This enum type specifies code creation mode. Sections are indicated in brackets:

    ConstantValueDescription
    Flags::FullMode0x000001(mode) All enums and variables are copied, enabling some flexibility in use.
    Flags::DebugMode0x000002(mode) fullMode + debug messages.
    Flags::CompactMode0x000004(mode) Only most needed methods, variables and enums are preserved. Code is as small, as possible, at the expense of loosing flexibility.
    Flags::Subclass0x000008converter creates web methods by subclassing QWebMethod (this option requires QWebService library to be present).
    Flags::Synchronous0x000010(synchronousness) Web methods will wait for reply and return with data.
    Flags::Asynchronous0x000020(synchronousness) Web methods will return instantly. Data availability will be announced by emitting a signal.
    Flags::StandardStructure0x000100(structure) Headers will be placed in <dir>/headers, sources in <dir>/sources, build system and .pro files in <dir>/.
    Flags::NoMessagesStructure0x000200(structure) Converter will not create web methods as separate classes, but use QWebMethod (this option requires QWebService library to be present).
    Flags::AllInOneDirStructure0x000400(structure) All generated files will be stored in one folder.
    Flags::Qmake0x001000(build system) qmake will be used.
    Flags::Cmake0x002000(build system) cmake will be used.
    Flags::Scons0x004000(build system) scons will be used.
    Flags::NoBuildSystem0x008000(build system) No build system files will be created.
    Flags::Http0x010000(protocol) http protocol will be used.
    Flags::Soap100x020000(protocol) SOAP 1.0 protocol will be used.
    Flags::Soap120x040000(protocol) SOAP 1.2 protocol will be used.
    Flags::Soap0x060000(protocol) Internall wrapper for all SOAP protocols. When used in app, will default to SOAP 1.2
    Flags::Json0x080000(protocol) JSON protocol will be used.
    Flags::Xml0x100000(protocol) XML protocol (non-SOAP) will be used.
    Flags::Rest0x200000(non-exclusive, protocol) Code for communicating with RESTful WSes will be generted. This can be combined with any other protocol flag.

    The Options type is a typedef for QFlags<Option>. It stores an OR combination of Option values.

    Member Function Documentation

    Flags::Flags ( Options options_ = Options( FullMode | Synchronous | StandardStructure | Qmake | Soap12 ), HttpMethod method_ = Post, bool forced = false, QObject * parent = 0 )

    Constructs the Flags object. All params are optional. If you want to set non-default suffixes, use setObjSuffix() and setMsgSuffix().

    Default values:

    • options_ :
      • fullMode
      • synchronous
      • standardStructure
      • soap12
      • qmake
    • method_ = POST
    • forced = false
    • parent = 0

    See also setFlags().

    void Flags::setForced ( bool forced )

    Sets the forced value to given one.

    When 'true', converter will delete old sources, and create a fresh copy every time it is run.

    See also forced().

    void Flags::setHttpMethod ( HttpMethod newMethod )

    Sets HTTP method flag to newMethod. Any previous setting is discarded.

    See also httpMethod().

    bool Flags::setHttpMethod ( const QString & newMethod )

    Sets HTTP method flag to newMethod. Any previous setting is discarded. Only valid values are accepted, but checks are not case sensitive. Therefore, you can for example pass "delete" or "DELETE".

    Returns true if succesfull.
